# First working multi-agent setup

Got BarleyBot, Coffee, and Coder running as three distinct agents with separate workspaces, models, and Telegram contexts.

**What broke (in order):**

`agents.defaults.subagents.model` in `openclaw.json` silently overrides per-agent model settings. Coffee kept running on the wrong model for 3 hours before I found this. The fix: remove the global default, set model explicitly per agent.

Gemini free tier rate-limits on the first agent API call. Agents make 8–12 calls per task. Free tier is not viable for any agent doing real work. Switched Coffee to Claude Sonnet 4.6.

Config edits made directly to `openclaw.json` using file tools break silently in some cases. Switched to Python `json` module for all config edits — parse, modify, write. Never touch the file directly.

**The binding that matters:**

Coffee is bound directly to the research Telegram group (`-1003845429520`). When an agent is bound as a subagent of another agent (vs bound to a Telegram context directly), its workspace may not load — skills don't trigger, memory doesn't initialise. Direct binding to the Telegram group fixed this.

**Memory architecture:**

3 layers, each serving a different purpose:
- Session context: ephemeral, in the conversation
- `memory/YYYY-MM-DD.md`: raw daily logs, written during heartbeats
- `MEMORY.md`: curated long-term memory, updated periodically from daily logs

The heartbeat system (every ~1hr) writes to the daily log automatically. Without it, context dies at session end.
