# Stable paths, deploy hook, Obsidian architecture

**The /tmp problem:**

Cron jobs that write to `/tmp/siva-builds/` fail in new sessions — `/tmp` is ephemeral. Moved both repos to `/Users/siva/.openclaw/repos/` — persistent across sessions and reboots. The daily wrap cron now uses `git -C /Users/siva/.openclaw/repos/siva-builds` for all git operations.

**Vercel deploy hook wired:**

`curl -s -X POST [hook-url]` triggers a Vercel rebuild. This fires at the end of the daily wrap cron — after the log entry is committed to siva-builds. Result: the site updates automatically every night without any manual step.

**Cases live from GitHub:**

The proof-of-work site no longer has hardcoded case data. `lib/cases.ts` fetches `siva-builds/projects/cases.json` at build time via GitHub raw URL. ISR (`revalidate: 3600`) means it re-fetches every hour. Adding or updating a case = edit `cases.json` + push = site updates automatically.

**Obsidian integration architecture:**

Two vaults detected: `Hive` and `Hive - Siva Second Brain`. The decision: keep OpenClaw workspace and Obsidian vault separate. Obsidian is human-facing knowledge capture. MEMORY.md is agent-facing distilled intelligence. The integration pattern: agent reads from Obsidian (one-way pull) → distilled insights go into MEMORY.md → future sessions use MEMORY.md without re-reading the vault. Obsidian stays clean. Agent memory stays lean.
