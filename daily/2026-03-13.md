## gstack — Role-Based Claude Code Commands

Spent the morning session studying Garry Tan's (YC President) open-source `gstack` repo — six slash commands that give Claude Code opinionated roles rather than running it in generic mode. The commands break down cleanly: `/plan-ceo-review` stress-tests whether you're building the right thing (10-star product framing), `/plan-eng-review` covers architecture + data flow + edge cases + test matrix, `/review` acts as a paranoid staff engineer hunting production bugs not style, `/ship` handles the full release loop (sync main → tests → push → PR), `/browse` gives Claude Code visual context (logs in, clicks around, screenshots), and `/retro` analyses commit history and shipping velocity.

The `/browse` command is the most interesting one architecturally — it solves the fundamental problem that Claude Code operates blind to the actual running UI. For Coder agent workflows this is a significant unlock: full QA in ~60 seconds without manual screenshot handoffs.

## Vayu App — Governance Architecture Mapped

Pulled up the Vayu repo (Next.js breathwork/meditation app) and mapped its full `.claude/commands/` governance layer. It has a genuinely rigorous release gate: `/ship` checks `MANIFEST.md` sprint state, runs walkthroughs, healthcheck, and `MEMORY.md` before any push is allowed. Supporting commands cover `/dev`, `/prd`, `/conductor`, `/healthcheck`, `/sync`, `/walkthrough`, `/test`, and `/init`. The sprint tracker, memory file, override mechanisms, and UI redesign guide are all present. Yesterday's work on that repo: Bellows Breath added, audio bugs fixed, `fix/session-audio-bugs` branch pushed.

## Safe Install Plan — gstack vs Vayu Conflict Resolution

The critical finding: both gstack and Vayu have a `/ship` command, but they do completely different things. Vayu's `/ship` is a rigorous release gate with multiple pre-flight checks; gstack's is a simple sync-test-push-PR flow. Claude Code project-level commands always shadow global ones inside that repo — so gstack's `/ship` would be invisible inside Vayu. That's the right behaviour. Safe install strategy: add all gstack commands globally to `~/.claude/commands/` *except* `/ship`, which stays Vayu-specific. No conflicts, full capability everywhere else.
