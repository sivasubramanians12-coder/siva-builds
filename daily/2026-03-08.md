## QMD Memory Index — Fixed and Automated

The QMD index had drifted 6 days behind. Ran `qmd update` + `qmd embed` manually — 13 new files ingested, 56 chunks embedded in 4 seconds on Apple M4 Metal. Index now covers 32 files (up from 19).

Root cause: no automated reindex was scheduled. Fixed by adding a new cron — "QMD Reindex" — running at 2:45 AM IST daily. Timing is intentional: picks up yesterday's memory files before the morning heartbeat and Daily Wrap crons fire. Implementation is a session:isolated cron so it doesn't pollute main session history.

Two open decisions flagged but not resolved: (1) memory decay policy — daily files currently accumulate indefinitely, proposal is monthly roll-up into MEMORY.md then delete raws; (2) MEMORY.md itself is stale at 121 lines, last updated Mar 2, needs a refresh pass.

## Paperclip Evaluated — Watch, Don't Adopt

Reviewed `paperclipai/paperclip` — a Node.js + React management layer that sits above agent frameworks. Key features: org charts for agents, per-agent monthly budget caps, heartbeat scheduling, audit logs, multi-company support. Upcoming: Clipmart (downloadable pre-built companies).

Verdict: overhead is too high at the current scale (Coffee + Coder + BarleyBot). The budget enforcement and audit log features become genuinely useful when running 10+ agents across multiple projects. Filed as "revisit at scale." The interesting pattern it surfaces: treating agents as employees with job descriptions, reporting lines, and cost centres — that framing is useful even without the software.
