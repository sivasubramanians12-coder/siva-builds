## Competitor Monitor — Architecture Finalised

The Competitor Monitor concept surfaced Wednesday via the ideas cron and is now scoped to a buildable spec:

**Stack:** `tavily_crawl` for web traversal (no scraper to maintain, handles JS rendering), Claude for signal classification, OpenClaw cron for weekly scheduling, Telegram for delivery. Config in a simple JSON file — competitor URLs, signal categories (product / pricing / campaign), delivery channel ID.

**Trigger:** Weekly, Monday 8 AM IST. Output: structured Telegram brief — new products spotted, pricing signals, campaign themes — per competitor.

**Why it ships fast:** `tavily_crawl` does all the heavy lifting. Claude's summarisation is one prompt. OpenClaw handles cron + delivery natively. Total novel code: ~80 lines of Python config + prompt logic. Effort: 2. Impact: 4.

HD India context: Luxaflex and Sunwood both have active web presence. This is currently a manual quarterly check at best. Automating it to weekly delivery costs less than half a day and converts a blind spot into a regular signal.

## Order Cascade Monitor — Remains Highest Priority

Still the top unbuilt item on the Ops OS stack. Impact 5, Effort 2 — that ratio is hard to beat. The spec: a morning agent that reads HD India's pipeline data, calculates cycle time delta vs. the 17→14 day target, identifies stalled orders by stage, and sends a structured Telegram briefing at 8 AM IST.

The blocker isn't technical — it's data access. The pipeline data currently lives in Excel. Before this tool can ship, the data layer needs a defined source: either a CSV export path or a live Sheets/Airtable integration. That's the Monday conversation.
